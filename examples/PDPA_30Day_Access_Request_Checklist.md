# PDPA 30-Day Data Access / Correction Request Checklist

**For Route A v1.1.0 (access_request_days=30)**

## Receipt & Logging
- [ ] Log request receipt date and requester identity verification
- [ ] Acknowledge receipt within reasonable time (e.g. 3-5 business days)
- [ ] Categorize: access, correction, or combined request

## Processing (within 30 days)
- [ ] Locate all relevant personal data across systems
- [ ] Verify requester is data subject or authorized representative
- [ ] Prepare response: provide data in intelligible form or confirm correction
- [ ] If refusing (limited grounds), provide written reasons and appeal info

## Response & Record
- [ ] Deliver response no later than 30 days from receipt
- [ ] Document any extensions (max +30 days with justification) and notify requester
- [ ] Update audit logs with request ID, actions taken, response date
- [ ] Retain request records for compliance audits (min 3 years recommended)

**PDPA Requirement**: Organizations must respond to access/correction requests within 30 days.
